Admin Feedback Analysis

Summary: Owners and admins requested VIP features (extra vote, custom name tags), group-scoped voting and role-based session control, and live UX improvements (live updates, persistent login, numpad PIN input, PIN toggle).

Feature list

  • VIP Double-Vote & Profile Tags — Owner: VIPs get two votes and can set a short display tag.
  • Group-Scoped Voting & Role Starters — Admin: Sessions scoped to groups; roles can start sessions for others.
  • Live Updates & Login UX — Owner: Live voting state, persistent login, numpad PIN input, disable PIN option.

Acceptance (examples)

  • VIP users begin a session with `votes_left`=2; non-VIP with 1.
  • Scoped sessions only accept votes from group members.
  • Clients reflect session state within 2–5s or via realtime subscriptions.

DB / Policies

  • Add `users.is_vip`, `users.display_tag`, `users.votes_left`.
  • Create `groups`, `group_memberships`, `voting_sessions`, and `votes` tables.
  • RLS: allow SELECT/INSERT/UPDATE for auth roles and membership checks.

Quick SQL (example)

-- Add VIP flag and votes_left
    ALTER TABLE public.users
        ADD COLUMN IF NOT EXISTS is_vip BOOLEAN DEFAULT false,
        ADD COLUMN IF NOT EXISTS display_tag TEXT NULL,
        ADD COLUMN IF NOT EXISTS votes_left INT DEFAULT 1;

    -- Groups
    CREATE TABLE IF NOT EXISTS public.groups (id TEXT PRIMARY KEY, name TEXT);
    CREATE TABLE IF NOT EXISTS public.group_memberships (user_id UUID REFERENCES auth.users(id), group_id TEXT REFERENCES public.groups(id), PRIMARY KEY (user_id, group_id));

    -- Sessions & votes
    CREATE TABLE IF NOT EXISTS public.voting_sessions (id UUID PRIMARY KEY DEFAULT gen_random_uuid(), title TEXT, scope TEXT DEFAULT 'public', created_by UUID REFERENCES auth.users(id), starts_at TIMESTAMPTZ, ends_at TIMESTAMPTZ);
    CREATE TABLE IF NOT EXISTS public.votes (id UUID PRIMARY KEY DEFAULT gen_random_uuid(), session_id UUID REFERENCES public.voting_sessions(id), user_id UUID, movie_id TEXT, display_tag TEXT, created_at TIMESTAMPTZ DEFAULT now());

Top next steps

  1. Run the SQL to add VIP and sessions tables.
  2. Implement atomic vote decrement for `votes_left`.
  3. Enable realtime subscriptions and update clients.
